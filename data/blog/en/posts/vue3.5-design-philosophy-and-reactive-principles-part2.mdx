# How `computed` works

```js
// example
const state = reactive({ name: "zoyi" });

const aliasName = computed(() => {
  console.log("getter runs");
  return "**" + state.name;
});

effect(() => {
  console.log("outer effect runs");
  console.log(aliasName.value);
});

state.name = "star zoyi";
```

## Initialization

1. `computed(getter)` returns a `ComputedRefImpl` instance `aliasName` with an inner `ReactiveEffect(getter, scheduler)`; `.value` is reactive.

```js
export class ComputedRefImpl {
  constructor(getter: () => any) {
    this.effect = new ReactiveEffect(getter, () => {
      //...
    });
  }
}
```

2. `effect(fn)` creates the outer effect and runs it.
3. Logs **outer effect runs**. Reading `aliasName.value` enters the inner effect’s `get value`.
4. If there is an `activeEffect` (outer `effect.run()`), the outer effect is tracked on `aliasName.dep`.

```js
get value() {
  // outer effect reads computed: record outer effect on this ref’s dep
  this.trackComputed();
  if (this._dirty === DirtyLevels.Dirty) {
    this._dirty = DirtyLevels.NoDirty;
    this._value = this.effect.run();
  }
  return this._value;
}

/** track “who depends on this computed” */
  private trackComputed() {
    if (!activeEffect) {
      return;
    }
    this.dep ??= createDep(() => {
      this.dep = undefined;
    }, "computed");
    trackEffect(activeEffect, this.dep);
  }
```

5. First time `_dirty` is dirty → clear dirty and run inner `effect.run()` (the computed getter runner).
6. `activeEffect` becomes the inner effect; getter runs, logs **getter runs**, reads `state.name`, inner effect collects `name` as dependency. Returns `name = zoyi`.
7. Getter returns `@zoyi`, cached in `aliasName._value`; `get value` finishes.
8. Logs **@zoyi**; outer `effect.run()` completes.

_Relationship:_

> dep of `state.name` → inner `ReactiveEffect` (computed scheduler).  
> `aliasName.dep` → outer effect (because it read `.value`).

## Update phase (Vue 3.4)

1. `state.name = "star zoyi"` triggers setter on `name`.

   ```js
   set(target, key, value, recevier) {
    let oldValue = target[key];
    let result = Reflect.set(target, key, value, recevier);

    // 只有新旧值不一样才会触发更新
    if (oldValue !== value) {
      trigger(target, key, value, oldValue);
    }

    return result;
   }
   ```

2. Value changed → `trigger` → inner effect’s scheduler.
   - By default it does not `run` immediately; only marks `_dirty`.
   - `triggerEffects(aliasName.dep)` → outer effect’s scheduler → outer `effect.run()` again.

```js
constructor(getter: () => any) {
  // 不在此构造函数里立即 run：首次访问 .value 时再求值，实现惰性。
  // scheduler：依赖变更时不立刻重算，只标脏并通知「读过我的人」去更新。
  this.effect = new ReactiveEffect(getter, () => {
    if (this._dirty === DirtyLevels.NoDirty) {
      this._dirty = DirtyLevels.Dirty;
    }
    if (this.dep) {
      triggerEffects(this.dep); // aliasName.dep
    }
  });
}
```

3. Logs **outer effect runs**, reads `aliasName.value` again.
   - `trackComputed()` records outer effect on `aliasName.dep` again.
   - `_dirty` is dirty → `this.effect.run()` → logs **getter runs**, reads new `state.name`, gets `@star zoyi`, caches, clears dirty.
4. Logs **@star zoyi**, update done.

## Note: Vue 3.5 changes computed’s update phase slightly

## Update phase (Vue 3.5)

1. `state.name = "star zoyi"` triggers setter.
2. Value changed → `trigger` → inner effect scheduler.
3. **Change:** run `refreshComputed` → if dirty, clear dirty first → `this.effect.run()` → log **getter runs**, read new name, cache `@star zoyi` into `_value`.

```js
constructor(getter: () => any) {
  this.effect = new ReactiveEffect(getter, () => {
    // 3.5 风格：先置脏并同步重算，再通知下游（顺序与官方包一致）
    this._dirty = DirtyLevels.Dirty;
    this.refreshComputed();
    if (this.dep) {
      triggerEffects(this.dep); // 再执行外层 effect.run
    }
  });
}

/**
 * 若当前为脏，则执行内层 effect（getter），更新 _value 并清脏。
 */
private refreshComputed() {
  if (this._dirty !== DirtyLevels.Dirty) {
    return;
  }
  this._dirty = DirtyLevels.NoDirty;
  this._value = this.effect.run(); // 先执行 getter
}
```

4. Then outer `effect.run` logs **outer effect runs**, reads `aliasName.value`.
   - `trackComputed()` again.
   - Value already recomputed; read straight from `_value`.
5. Logs **@star zoyi**, done.

```js
get value() {
  // 收集计算属性（aliasName）的依赖，再保证缓存最新
  this.trackComputed();
  this.refreshComputed(); // _dirty 为不脏直接返回
  return this._value; // 已经计算过新的属性了，直接从_value中获取
}
```

# How `watch` works

```js
watch(
  { state.name }, // source
  (prev, next, onCleanup) => { //cb
    console.log("callback fired")

    onCleanup(() => {
      console.log("cleanup side effect");
    });
  },
  {
    immediate: false, // 立即执行一次
    deep: false // 是否深度监听
  });
```

**When `source` changes, `cb` runs.**

Implementation idea: `source` (must be reactive) is tracked by an effect; when scheduler runs, call `cb` with old/new values.

```js
function watch(source, cb, options?) {
  const { immediate = false, deep = false } = options;
  const getter = createWatchGetter(source, deep);

  let oldValue;
  let cleanup;

  // 初始化 effect，值变化时进行更新操作
  const _effect = new ReactiveEffect(getter, () => {
    const newValue = _effect.run(); // 获得最新的值

    if (cleanup) {
      cleanup();
      cleanup = undefined;
    }

    cb(newValue, oldValue, (fn) => {
      cleanup = fn;
    });

    oldValue = newValue;
  });

  oldValue = _effect.run();

  // 立马执行一次 cb
  if (immediate) {
    cb(oldValue, undefined, (fn) => {
      cleanup = fn;
    });
  }

  return () => {
    if (cleanup) {
      cleanup();
      cleanup = undefined;
    }
    stopEffect(_effect);
  };
}
```

**`createWatchGetter` turns `source` into a runnable getter for dependency collection.**

> Supported `source` types: `ref`, `reactive`, arrays (iterate), functions.

```js
function createWatchGetter(source: unknown, deep: boolean): () => unknown {
  if (isRef(source)) {
    return () => (source as { value: unknown }).value;
  }
  if (typeof source === "function") {
    return source as () => unknown;
  }
  if (isArray(source)) {
    return () =>
      (source as unknown[]).map((s) => {
        if (isRef(s)) {
          return (s as { value: unknown }).value;
        }
        if (typeof s === "function") {
          return (s as () => unknown)();
        }
        return s;
      });
  }
  if (isReactive(source)) {
    // deep 为 true 则深度监听，否则只监听一层
    const maxDepth = deep ? undefined : 1;
    return () => traverse(source, maxDepth);
  }
  return () => source;
}
```

**Cleanup: `onCleanup` is the third argument to the callback; it registers a function to run before the next callback or when stopping.**

```js
// example
watch(
  () => state.id,
  (id, oldId, onCleanup) => {
    let cancelled = false;
    onCleanup(() => {
      cancelled = true;
    });

    fetch(`/api/user/${id}`).then((res) => {
      if (!cancelled) {
        state.user = res;
      }
    });
  },
);
```

**Stopping: watch return value can call `stopEffect`.**

```js
/**
 * 停止副作用：从各 dep 中移除并清空依赖列表，之后不再被 trigger。
 */
export function stopEffect(effect: ReactiveEffect) {
  if (!effect.active) {
    return;
  }
  effect.active = false; // 激活状态改为 false
  const deps = effect.deps;
  for (let i = 0; i < deps.length; i++) { // 并清理 effect 上的 deps
    cleanDepEffect(deps[i], effect);
  }
  effect.deps.length = 0;
}
```

**Options API: `flush`**

- `pre` (default): run shortly after in the same tick (often still microtask), usually before component re-render—read old DOM or tweak other state first.
- `post`: after DOM updates—good when you need layout/measurement on updated DOM.
- `sync`: run callback synchronously right when deps update—no microtask batching, no waiting for render phase.
